enlace Descarga de jQuery

Copias comprimidas y sin comprimir de archivos jQuery están disponibles. Se utiliza mejor el archivo sin comprimir durante el desarrollo o la depuración; el archivo comprimido ahorra ancho de banda y mejora el rendimiento en la producción. También puede descargar un archivo Sourcemap para su uso cuando se depura con un archivo comprimido. El archivo de mapa es no necesario para los usuarios ejecutar jQuery, sólo mejora la experiencia del depurador del desarrollador. A partir de jQuery 1.11.0 / 2.1.0 del // # sourceMappingURL comentario está no incluido en el archivo comprimido.

Para descargar estos archivos a nivel local, haga clic en el vínculo y seleccione "Guardar como ..." en el menú.

La línea 1.x jQuery tuvo grandes cambios como de jQuery 1.9.0. Lea la Guía de actualización de jQuery 1.9 y la 1.9 liberación entrada en el blog de ​​jQuery para más información.

vincular jQuery 2.x

jQuery 2.x tiene la misma API como jQuery 1.x, pero no es compatible con Internet Explorer 6, 7, u 8 . Todas las notas de la Guía de jQuery 1.9 de actualización se aplican aquí también. Desde IE 8 es todavía relativamente común, se recomienda utilizar la versión 1.x a menos que esté seguro de no hay IE 6/7/8 usuarios visitan el sitio. Por favor, lea la versión 2.0 toma nota cuidadosamente.

jQuery está registrada como un paquete con Bower . Puede instalar la última versión de jQuery con el comando:

Esto instalará jQuery para directorio de instalación de Bower, siendo el valor por defecto bower_components . Dentro bower_components / jQuery / dist / usted encontrará una versión sin comprimir, un comunicado de comprimido, y un archivo de mapa.

El paquete de jQuery Bower contiene archivos adicionales además de la distribución por defecto. En la mayoría de los casos se puede hacer caso omiso de estos archivos, sin embargo si usted desea descargar la versión por defecto por sí mismo que usted puede utilizar para instalar Bower jQuery de una de las direcciones URL en lugar del paquete registrada anteriormente. 1

vincular jQuery Plugin Migrar

Hemos creado el plugin de jQuery Migrar para simplificar la transición desde versiones anteriores de jQuery. El plugin restaura Características en desuso y comportamientos para que el código más antiguo todavía se ejecute correctamente en jQuery 1.9 y versiones posteriores. Utilice el desarrollo sin comprimir versión para diagnosticar problemas de compatibilidad, generará advertencias en la consola que se puede utilizar para identificar y solucionar problemas. Use la producción comprimido versión para simplemente arreglar los problemas de compatibilidad sin generar advertencias de la consola.

vincular Construye jQuery Pre-Release

El equipo de jQuery está constantemente trabajando para mejorar el código. Cada comprometen a la cesión temporal Github genera una versión de trabajo en curso del código que actualizamos en el jQuery CDN. Estas versiones son a veces inestable y no es adecuado para los sitios de producción. Recomendamos que se utilizan para determinar si un error ya ha sido fijo al informar errores contra versiones publicadas, o para ver si se han introducido nuevos errores. Hay dos versiones, una para la rama 1.x (soporta IE 6/7/8) y uno para la rama 2.x (no para el uso con el viejo IE).

enlace Usando jQuery con un CDN

CDN puede ofrecer una ventaja de rendimiento al acoger jQuery en servidores repartidos por todo el mundo. Esto también ofrece la ventaja de que si el visitante de su página web ya ha descargado una copia de jQuery de la misma CDN, no tendrá que volver a descargar.

Enlace a CDN de jQuery proporcionada por MaxCDN

Para utilizar el jQuery CDN, simplemente hacer referencia al archivo directamente desde http://code.jquery.com en la etiqueta de script:

1
2
 
< guión  src = "//code.jquery.com/jquery-1.11.0.min.js" ></ script de >
< guión  src = "//code.jquery.com/jquery-migrate-1.2.1.min.js" ></ script de >

A partir de jQuery 1.9, archivos Sourcemap están disponibles en el jQuery CDN también. Si se incluyen archivos comprimidos directamente de la CDN como se muestra arriba, los navegadores-Sourcemap conscientes como Google Chrome usarán cuando se habilita el apoyo Sourcemap.

Para ver todos los archivos y versiones disponibles, visite http://code.jquery.com

enlace Sobre el Código

jQuery es proporcionado bajo la licencia MIT .

El código está alojado y se desarrolló en el repositorio de GitHub jQuery . Si usted ha manchado algunas zonas de código que se pueden mejorar, por favor siéntase libre de hablar de ella en el desarrollo de jQuery Foro Core . Si desea participar en el desarrollo de jQuery, leer nuestra sitio colaborador para más información.

Para encontrar y descargar los plugins desarrollados por colaboradores jQuery, por favor visite el sitio de Plugins . Plugin autores son responsables del mantenimiento de sus plugins. Comentarios sobre plugins debe ser dirigida al autor del plugin, no el equipo de jQuery.

enlace Build de Git

Nota: Para utilizar sólo la versión más reciente trabajo en curso de jQuery, por favor intente la versión preliminar de jQuery se ha descrito anteriormente.

Todo el código fuente se mantiene bajo control de revisiones de Git, que se puede navegar en línea. El repositorio de README tiene más información sobre la construcción y las pruebas de su propia jQuery, así como instrucciones sobre cómo crear una generación personalizada que excluye a algunas API para reducir el tamaño del archivo.

Si usted tiene acceso a Git, puede conectarse al repositorio aquí:
